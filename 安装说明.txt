================================================================================
电力交易员刷题 — 本地使用说明（打包版 / 源码运行）
================================================================================

【一、使用已打包好的程序（推荐）】

1. 解压收到的 zip，得到文件夹「电力交易员刷题」。
2. 将题库 Excel 文件放到该文件夹内，文件名与默认一致：
   电力交易员中级工题库2025(2).xlsx
   （若文件名不同，需用源码方式在 exam.py 里改默认文件名，或向开发者索取对应配置。）
3. 双击运行（必须在「dist」文件夹里，不要用「build」）：
   - 正确：…/刷题软件/dist/电力交易员刷题/电力交易员刷题
   - 错误：…/刷题软件/build/… 下的同名文件（仅为打包中间产物，会报 Python3 找不到）
   - Windows：dist\电力交易员刷题\电力交易员刷题.exe
   - macOS：dist/电力交易员刷题/电力交易员刷题
4. 若未自动打开浏览器，请手动访问：http://127.0.0.1:5001/
5. 关闭程序：关闭弹出的黑色命令行窗口，或在窗口里按 Ctrl+C。
6. 学习进度保存在同目录下的 question_progress.csv，勿删以免丢失进度。

macOS 若提示「无法打开」：可在「系统设置 - 隐私与安全性」中允许，或右键 - 打开。

【网页打不开时】

1. 确认黑色命令行窗口还开着（关掉窗口即停止服务）。
2. 地址必须是 http://127.0.0.1:5001/ （默认端口为 5001，不是 5000）。请从地址栏核对，不要沿用旧书签里的 5000。
3. 若终端提示「端口已被占用，已改用 5xxx」，请用终端里打印的新端口号访问。
4. 先访问 http://127.0.0.1:5001/health 若能看到 {"ok":true} 说明服务正常，再打开首页。
5. 不要重复运行多份程序，否则会抢端口；关掉多余的窗口再试。

--------------------------------------------------------------------------------
【三、在线网页版（GitHub Pages，可选）】

桌面打包与本地 python web_app.py 的方式不变。若要把同一题库发布成公开网页（使用
https://<你的用户名>.github.io/<仓库名>/ 这类地址）：

1. 在电脑上安装 Python 与依赖（pip install flask pandas openpyxl 等，与本地刷题相同）。
2. 项目根目录执行：  python3 export_bank_for_pages.py
   会在 docs/questions.json 写入题库数据（与当前 question_progress.csv / Excel 一致）。
3. 用 Git 初始化仓库，将项目（含 docs/）推送到 GitHub。
4. 打开仓库 Settings → Pages → Build and deployment：
   Source 选「Deploy from a branch」，Branch 选 main（或你的默认分支），文件夹选「/docs」保存。
5. 等待几分钟后用浏览器访问站点。更新题库后需重新执行 export 并推送新的 docs/questions.json。

说明：在线版纯静态页面，进度保存在访客浏览器的 localStorage，与桌面版 CSV 不互通；
换浏览器或清除数据会丢进度。
